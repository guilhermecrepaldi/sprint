# Contributing

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feat/description`)
3. Commit your changes using conventional commits
4. Push to the branch (`git push -u origin feat/description`)
5. Open a Pull Request

## Commit Guidelines

Use conventional commits: feat:, fix:, docs:, refactor:, perf:, test:, chore:, security:

## Code Standards

- Follow existing code style (Kotlin/Python)
- Write tests for new features
- Update documentation when changing behavior
